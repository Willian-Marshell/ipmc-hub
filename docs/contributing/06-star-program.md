# 开源之星计划

> **状态**：本计划为 `draft`，规则细节待与微专业老师对齐后定稿。
> 早期贡献者的所有付出都会被追溯计入。

## 为什么要有激励

写文档、做 review、维护工具链都需要时间。让贡献被看见、被记住、被奖励，是这个项目能跑起来的根基。

## 评选周期

- **每学期**评选一次
- **每学年**额外评选一次"年度开源之星"

## 计分维度

每条贡献都对应一个**积分**，按四个维度加权：

| 维度 | 说明 | 权重 |
|---|---|---|
| **数量** | 合入的 PR 数 | × 1 |
| **质量** | 栏目主编评分（1–5 分） | × 2 |
| **影响** | 浏览量、引用次数、issue 解决数 | × 1 |
| **稀缺性** | 填补空白板块的内容加权 | × 1.5 |

具体打分细则由当届运营组在每学期初公示，并记录在 `docs/contributing/star-rubric.md`（待补）。

## 奖励形式

奖励组合可能包括（最终以微专业每学期通告为准）：

- **荣誉**：站点首页"开源之星"轮播 + GitHub README 头衔
- **资源**：大模型 API 额度报销（如 Claude / DeepSeek API 充值）
- **奖金/礼品**：由微专业经费或赞助提供
- **推荐**：栏目主编、维护者优先从开源之星中选拔

## 谁来评

每届成立 3–5 人**运营组**：

- 1 名当届微专业老师 / 助教
- 2–3 名栏目主编
- 1 名上届开源之星

运营组负责打分、公示、发奖。打分过程留痕在 issue 中可追溯。

## 防作弊

- 自我互评不计分（同 PR 作者与 reviewer 不能是同一人）
- 同一内容拆成多 PR 不会获得更高分
- 灌水 PR（无实质内容、刷字数）由栏目主编打 0 分并不计入

## 历届开源之星

当届评选完成后，结果会发布在 `docs/hall-of-fame/<year>.md`（待建）。
